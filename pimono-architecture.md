## 📦 各包职责详解

### 核心层 (Core)

| 包名 | 主要职责 | 关键设计 |
|------|---------|---------|
| **pi-ai** | 统一 LLM API | 事件流抽象、多提供商适配、Context 序列化 |
| **pi-agent-core** | Agent 运行时 | 事件循环、状态机、工具执行、steering 机制 |

### 表现层 (Presentation)

| 包名 | 主要职责 | 关键设计 |
|------|---------|---------|
| **pi-tui** | 终端 UI | 差异渲染、同步输出、组件化架构 |
| **pi-web-ui** | Web UI | 聊天界面组件、可复用 |

### 应用层 (Applications)

| 包名 | 主要职责 | 构建于 |
|------|---------|--------|
| **pi-coding-agent** | 交互式编码助手 | pi-agent-core + pi-tui |
| **pi-mom** | Slack 消息代理 | pi-agent-core |
| **pi-pods** | GPU Pod 管理 CLI | pi-tui |

## 🎯 关键设计决策

### 1. 流式事件优先
所有 LLM 交互都是事件流，而非请求-响应。这让 UI 可以实时更新，支持打字效果、思考过程展示。

### 2. Context 可序列化
对话上下文是 Plain JSON，可以：
- 保存/恢复会话
- 在不同提供商间传递
- 实现跨模型协作

### 3. 事件驱动的 Agent 架构
Agent 不是一次函数调用，而是一个持续运行的状态机，支持：
- 工具调用循环
- 运行时 steering（打断/干预）
- 流式工具结果

### 4. 差异渲染 TUI
终端 UI 不是每次都清屏重绘，而是：
- 计算前后差异
- 只更新变化的部分
- 使用 CSI 2026 同步输出防闪烁

### 5. 清晰的类型分层
```
AgentMessage (业务层)
    │ convertToLlm()
    ▼
Message (LLM 层)
    │ provider-specific transform
    ▼
Provider-specific format (OpenAI/Anthropic/Google)
```
